# zarg4n Career Overhaul 0.2.0

Author / Yapımcı: zarg4n

## Türkçe

0.2.0, genç oyuncu gelişimini yalnız reyting tavanına değil sezon performansına bağlayan ilk kararlı çekirdek sürümdür. Forma sayısı, maç puanı, gol, asist, clean sheet ve kurtarışlar; pozisyona ve örnek büyüklüğüne göre kontrollü biçimde değerlendirilir. Gelişim 27–28 yaş prime dönemini kapsar, sezonluk potansiyel değişimi sınırlıdır ve oyuncular iki sezonda süperstara dönüşmez.

Fiziksel profil, kişilik ve rol eğilimi deterministiktir. Oyuncuların tamamı aynı kalıba girmez; gelişim sonucunda uygun PlayStyle adayları oluşabilir. Kalıcı işlem günlüğü yarım kalan sezon sonu işlemlerini güvenli biçimde yeniden dener.

**Fresh save zorunludur:** Modu kurduktan sonra yeni bir Manager Career oluştur. Eski kariyerler ve eski runtime state dosyaları desteklenmez.

Bu sürüm oyun içi transfer ikna ekranı, yeni HUD, yeni basın/mesaj seçenekleri, isim havuzu, Create-a-Club seçici, yedek kulübesi sayısı değişikliği veya “pozisyonunda kal” maç puanı düzeltmesi içermez. Güvenli ve belgelenmiş bir API olmadan gameplay ya da bellek yaması kullanılmadı.

## English

0.2.0 is the first stable core release that ties player development to seasonal performance instead of rating ceilings alone. Appearances, match rating, goals, assists, clean sheets and saves are evaluated conservatively by position and sample size. Development covers the 27–28 prime window, potential changes remain capped, and players will not become superstars in two seasons.

Physical identity, personality and role tendencies are deterministic without forcing every player into the same template. Eligible development can produce appropriate PlayStyle candidates. A durable transaction journal safely retries interrupted end-of-season work.

**A fresh save is mandatory:** create a new Manager Career after installing the mod. Existing careers and legacy runtime state files are unsupported.

This release does not include an in-game transfer-persuasion screen, a new HUD, new press/message choices, expanded name pools, a Create-a-Club selector, bench-size changes or a “stay in position” rating fix. No gameplay or memory patch was used without a safe, documented API.
